# Models Architecture

This directory contains the core PyTorch model implementations for our multimodal earthquake forecasting pipeline.

## File Structure

### `safenet_embeddings.py`
This file contains a reproduction of the embedding generation pipeline from the original **SafeNet** paper. It handles the initial processing and feature extraction from the two primary data modalities:
* **Catalog Data**: Encoded into dense temporal representations.
* **Map Data**: Processed through a 3D Convolutional spatial block.

The embeddings generated by these modules act as the foundational inputs for the full architectures defined in `spatial_models.py`.

### `spatial_models.py`
This file contains the complete, end-to-end model architectures. Both models defined here utilize the base embedding logic from `safenet_embeddings.py`.

1. **`SafeNetFull`**: A faithful reproduction of the original SafeNet paper's complete model architecture.
   - Fuses the catalog and map embeddings.
   - Uses an LSTM to capture temporal dynamics independently for each geographic region.
   - Applies a Vision Transformer (ViT) encoder to share information across the spatial regions.
   - Outputs per-patch magnitude class predictions.

2. **`SeiSM`**: Our custom architecture that completely replaces the LSTM and ViT blocks with a State Space Model (**Mamba**).

---

## How our SeiSM Works

1. **Embed**: Generate catalog and map embeddings → `(B, T, P, 64)` + a global token `(B, T, 32)`.
2. **Flatten**: Collapse all `P` patches into one vector per timestep and concatenate the global token → `(B, T, P*64+32)`.
3. **Mamba**: Project to `d_model` and pass through stacked residual Mamba blocks, modeling the temporal evolution of the entire spatial grid at once.
4. **Classify**: Take the last timestep's hidden state and project it to `P × num_classes` to produce per-patch magnitude predictions.
